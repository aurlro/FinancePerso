Page : Dépenses à valider

Objectif :
Permettre à l’utilisateur de valider ou corriger les catégories proposées par l’IA
et d’enseigner progressivement l’application.

Contenu :
- Liste des dépenses non catégorisées ou incertaines
- Pour chaque dépense :
  • Libellé
  • Montant
  • Source (perso / joint)
  • Proposition IA (catégorie, confiance)
  • Indication abonnement potentiel (si détecté)
- Actions rapides :
  • Valider la catégorie
  • Modifier la catégorie (liste + champ libre)
  • Créer une nouvelle catégorie

UX :
- Validation en 1 clic
- Feedback immédiat après action
- Sensation de “dialogue” avec l’IA

Style :
- Data-first
- Accent couleur uniquement pour attirer l’attention IA
