📋 Référentiel des Use Cases (User Stories & Logique)

UC-01 | Data Pipeline processor.py

Nettoyage et Catégorisation Automatique

"En tant qu'utilisateur, je veux que mes transactions brutes soient transformées en noms de commerçants lisibles et catégorisées automatiquement afin d'avoir une vue claire de mes dépenses sans effort."

Flux logique :

  1. Upload du CSV (app.py).
  2. Extraction Pandas et passage au dictionnaire Regex (processor.py).
  3. Si inconnu : appel API LLM avec schéma de sortie strict (ex: `{"merchant": "SNCF", "category": "Transport"}`).
  4. Sauvegarde en base (database_manager.py).
🔗 OpenAI Structured Outputs 🔗 Pandas Regex Extract
UC-02 | User Interface app.py & database.py

Correction Manuelle (Active Learning)

"En tant qu'utilisateur, je veux pouvoir corriger manuellement une mauvaise catégorisation directement dans un tableau, afin que mes graphiques soient justes et que l'application apprenne de ma correction."

Flux logique :

  1. Affichage des données via st.data_editor.
  2. L'utilisateur modifie la cellule "Catégorie".
  3. L'événement on_change déclenche une mise à jour en BDD.
  4. Mise à jour du dictionnaire local pour les futures transactions similaires.
🔗 Streamlit Data Editor 🔗 Streamlit Session State
UC-03 | Analytics Engine processor.py

Détection d'Abonnements (Charges Fixes)

"En tant qu'utilisateur, je veux que l'application identifie automatiquement mes prélèvements récurrents afin de calculer précisément mon 'Reste à vivre' mensuel."

Flux logique :

  1. L'algorithme groupe les dépenses par "Marchand" sur les 3 derniers mois.
  2. Si variance de montant < 5% et écart temporel ≈ 30 jours ➔ Flag is_sub = True.
  3. Soustraction automatique de ces montants du revenu pour isoler le reste à vivre.
🔗 Pandas GroupBy Aggregation
UC-04 | KPI & Alerting app.py

Suivi du "Cash-Burn" (Vitesse de dépense)

"En tant qu'utilisateur, je veux visualiser immédiatement si je dépense mon argent plus vite que le mois dernier, afin d'ajuster mon comportement avant la fin du mois."

Flux logique :

  1. Calcul de la somme cumulée des dépenses à J-courant pour le mois N et le mois N-1.
  2. Affichage d'un KPI dynamique avec flèche rouge/verte (st.metric).
  3. Rendu d'un graphique linéaire (Plotly) superposant la courbe de vélocité des deux mois.
🔗 Streamlit st.metric 🔗 Plotly Cumulative Lines
UC-05 | GenAI Advisory app.py & processor.py

Simulateur d'Épargne "What-If"

"En tant qu'utilisateur, je veux simuler des réductions sur des catégories spécifiques (ex: Restaurants) et recevoir un conseil textuel sur l'impact pour mon épargne cible."

Flux logique :

  1. Interface avec sliders (st.slider) pour modifier l'enveloppe budgétaire.
  2. Recalcul dynamique de l'épargne potentielle.
  3. Envoi d'un prompt au LLM intégrant le Delta :
    "L'utilisateur a réduit ses loisirs de 50€. Rédige un encouragement."
  4. Affichage de la recommandation générée.
🔗 Streamlit Sliders 🔗 OpenAI Chat API
UC-06 | Reporting processor.py

Génération du Rapport Synthetique Mensuel

"En tant qu'utilisateur, je veux obtenir un bilan écrit (façon banquier privé) résumant mes tops dépenses et mes écarts budgétaires à la fin du mois."

Flux logique :

  1. Extraction des "Top 3 catégories" et des variations vs Mois précédent.
  2. Assemblage des données dans un Prompt structuré (Data-to-Text).
  3. Le LLM rédige un compte-rendu formaté en Markdown.
  4. Affichage via st.markdown et bouton d'export optionnel.
🔗 Streamlit Markdown 🔗 Pandas nlargest (Top 3)
